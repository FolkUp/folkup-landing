## PRIMARY rollback — symlink switch к previous release (~30s RTO)

Атомарный `ln -sfn` (per AGIL-106 atomic fix).

```bash
# SSH к VPS
ssh ${VPS_USER}@${VPS_HOST}

# Find previous release
ls -1t /home/deploy/sites/folkup-landing/releases/ | head -10
# (Should show last 5 release timestamps + current symlink target)

# Identify which release к rollback to:
readlink /home/deploy/sites/folkup-landing/current
# e.g. /home/deploy/sites/folkup-landing/releases/20260530_120000

# Atomic switch к previous (e.g. 20260518_100000):
PREVIOUS_RELEASE="20260518_100000"  # adjust к actual previous
ln -sfn /home/deploy/sites/folkup-landing/releases/${PREVIOUS_RELEASE} \
        /home/deploy/sites/folkup-landing/current

# Restart container OR reload nginx (per deploy script logic)
if docker ps --format '{{.Names}}' | grep -q '^folkup-landing$'; then
  docker compose -f /home/deploy/folkup-landing/docker-compose.yml restart folkup-landing
else
  sudo nginx -t && sudo systemctl reload nginx
fi

# Verify
curl -sI https://folkup.app/ | grep -i etag
# Should match previous release etag
```

**RTO:** ~30 sec. **RPO:** zero (previous release файлы intact на VPS).

If previous release directory destroyed OR backups directory required.

